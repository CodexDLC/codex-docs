# codex-django Migration Guide

## Status

`planned`

## Target standard

`Docs Standard v1.0`

## Current state

One of the strongest current implementations and a likely reference source for the landing standard, but it still needs to conform explicitly to the shared contract.

## Must-have changes

- Normalize naming and support blocks to the standard contract
- Align README and PyPI to the docs-first model
- Ensure related-library blocks and migration links follow the canonical pattern
- Keep the site as a reference implementation for runtime-oriented packages

## Optional improvements

- Use the site to validate shared reusable landing components
- Add clearer standard-compliant version support messaging

## Release plan

- Release type: `documentation-only`
- Recommended bump: patch
