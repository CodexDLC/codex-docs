# Ecosystem Roadmap

## Phase 1: Establish governance baseline

- Publish `Docs Standard v1.0`
- Publish baseline ADRs
- Freeze the initial shell, naming, and versioning contract

## Phase 2: Normalize current repositories

- Bring landing pages to one canonical shape
- Normalize README and PyPI package surfaces
- Remove theme, naming, and version-label drift

## Phase 3: Launch shared ecosystem hub

- Publish the ecosystem landing
- Add library catalog and recommended paths
- Link every package docs site into one coherent map

## Phase 4: Introduce branded hosting

- Move from GitHub Pages identity to branded domain strategy
- Use one business site and one docs-focused domain model
- Preserve redirects from legacy URLs

## Phase 5: Scale for future libraries

- Require metadata and templates for all new libraries
- Make standards and migration review part of release governance
