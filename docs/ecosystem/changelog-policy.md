# Changelog Policy

## Scope split

Changelog and roadmap operate at two levels:

- library level for package-specific release history;
- ecosystem level for cross-library highlights and governance changes.

## Library changelog

Each package repository remains the canonical owner of its own changelog.

Required:

- documentation-only releases must be labeled clearly;
- breaking documentation changes must link to migration guides;
- changelog placement must be predictable in the library navigation.

## Ecosystem changelog

The hub does not duplicate library changelogs in full.

It may publish:

- standard releases;
- major ecosystem-wide docs transitions;
- curated highlights when multiple libraries change together.

## Roadmap policy

- Ecosystem roadmap lives in the hub
- Library roadmap is optional and must follow the standard if present
- Roadmap pages describe intended direction, not release history
