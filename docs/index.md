# Codex Docs Platform

`codex-docs` is the canonical documentation governance repository for the Codex ecosystem.

<div class="codex-callout" markdown="1">
It owns the ecosystem landing, documentation standards, shared templates, ADRs, and migration backlog. It does not replace the per-library documentation owned by each package repository.
</div>

## What lives here

- Ecosystem-level entrypoint and information architecture
- `Docs Standard v1.0`
- Shared rules for landing pages, README/PyPI, API reference, versioning, changelog, roadmap, and cross-links
- Decision records that explain why the model was chosen
- Migration plans for all current library repositories

## Recommended entry paths

- Start with [Libraries](libraries/index.md) to see the current ecosystem map
- Read [Standards](standards/index.md) to apply or review the canonical contract
- Use [Templates](template-pages/library-landing.md) when bringing a library into compliance
- Track rollout from [Migrations](migrations/index.md)

## Governance baseline

- Standard version: `Docs Standard v1.0`
- Canonical ownership model: `hybrid hub`
- Canonical version model: `latest + major lines`
- Canonical landing source of truth: `docs first`
