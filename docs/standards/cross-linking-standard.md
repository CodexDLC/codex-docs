# Cross-linking Standard

## Purpose

Cross-links must help users move between ecosystem layers without guessing relationships.

## Required cross-links

Every library landing page must link to:

- related libraries;
- its own API reference;
- changelog;
- migration guides if relevant;
- the ecosystem hub when published.

## Metadata-driven model

Cross-links should be derived from library metadata, not scattered ad hoc in arbitrary pages.

## Related libraries block

The block must explain the relationship, not just list package names.

Example relationships:

- foundation dependency;
- infrastructure dependency;
- runtime companion;
- higher-level integration layer.
