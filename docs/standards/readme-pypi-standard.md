# README and PyPI Surface Standard

## Purpose

README and PyPI are short-form package surfaces. They should not compete with the docs landing page.

## Required blocks

Every package-facing surface must include:

- package purpose;
- install instructions;
- one short quick-start path;
- key capabilities or modules;
- link to full docs;
- link to API reference;
- link to changelog;
- ecosystem position.

## Canonical behavior

- README and PyPI are concise
- Docs remain the canonical long-form explanation
- The same narrative order should be used across all packages

## Drift prevention

When docs landing changes materially, README and PyPI must be reviewed in the same change set or tracked explicitly in migration work.
