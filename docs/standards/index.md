# Docs Standard v1.0

This section is the canonical standard for the Codex documentation platform.

## Scope

The standard governs:

- ecosystem landing;
- per-library docs shell;
- library landing pages;
- README and PyPI package surfaces;
- API reference placement and naming;
- versioning, changelog, roadmap, and cross-links;
- migration tracking and exception handling.

## Normative rule

Library repositories must follow this standard unless an approved exception is recorded in:

- an ADR in `codex-docs`; or
- a migration page with explicit approval and rationale.

## Governance model

- The standard changes only through PRs in `codex-docs`
- Templates are normative defaults, not informal examples
- Deviations must cite an ADR or approved exception
- Every migration page must expose a status: `planned`, `in progress`, or `complete`
