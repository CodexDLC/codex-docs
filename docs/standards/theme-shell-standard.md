# Theme and Shell Standard

## Purpose

The user must experience one documentation system, not a family of related but visually drifting sites.

## Required shell contract

- Material for MkDocs
- Shared palette policy
- Shared dark and light toggle behavior
- Shared feature flags for navigation, search, code copy, and breadcrumbs
- Shared CSS override layer
- Shared callout and CTA patterns for landing pages

## Palette policy

- Default ecosystem palette: teal primary with cyan accent
- Exceptions require approval and should be rare
- Dark and light modes must both exist and behave the same way across sites

## Disallowed drift

- Different toggle semantics
- Arbitrary color families per repo
- Different top-level nav feature sets without strong justification
